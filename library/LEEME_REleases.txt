OBJETIVO:
- Tu APP sigue en Firebase Hosting
- Los PDFs se descargan desde GitHub Releases (aguanta mucho trafico)
- Las caratulas se quedan en Firebase (son livianas)

DATOS CLAVE (docs oficiales):
- GitHub Releases: cada archivo < 2 GiB y no hay limite de bandwidth usage para assets. (GitHub Docs)
- Por eso es ideal para descargas masivas.

PASO A PASO (UNA VEZ):
1) En GitHub: Repo Nclexpass/NCLEX1.2
2) Ve a: Releases -> "Draft a new release"
3) Tag: books
4) Sube TODOS tus PDFs como "release assets" (arrastrar/soltar)
   IMPORTANTE: el nombre del PDF debe coincidir EXACTO con el nombre en library/files.
   Ejemplo: Farmacologia_Pearson.pdf

PASO A PASO (CADA VEZ QUE AGREGAS LIBROS):
A) Copia PDF a: library\files\ (solo para que el script lea nombres y genere titulos)
B) Copia cover (jpg/png) a: library\covers\
C) Sube el PDF a GitHub Releases (tag books) como asset (mismo nombre)
D) Doble click: Actualizar_Biblioteca_Releases.bat
   -> genera catalog.json con links a GitHub Releases
   -> hace firebase deploy (publica app + covers + catalog)

RECOMENDADO (para no gastar cuota de Firebase con PDFs):
Edita firebase.json y agrega library/files a "ignore" dentro de hosting:

"hosting": {
  "public": ".",
  "ignore": [
    "firebase.json",
    "**/.*",
    "**/node_modules/**",
    "library/files/**"
  ],
  "rewrites": [{ "source": "**", "destination": "/index.html" }]
}

Listo.
